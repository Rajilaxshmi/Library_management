# Library_management

Overview

Library Management System (LMS) is a console-based application developed in C to automate and simplify library operations. The system helps librarians efficiently manage books, users, issue/return transactions, and maintain accurate records using file handling for persistent data storage.

Features

* Add new books to the library catalog
* Update existing book details
* Remove books from the system
* Search books by ID, title, or author
* View all available books
* Issue books with automatic issue and due date generation
* Return books and calculate late fines
* Track all issued books and their status
* Save and retrieve records using file handling

Technologies Used

* C Programming
* Structures
* Linked Lists
* File Handling
* Date and Time Functions
* Modular Programming

Objectives

* Automate library management operations
* Reduce manual record-keeping
* Improve accuracy and efficiency
* Maintain persistent book and transaction records

Key Functionalities

Book Management

* Add, update, delete, and search books
* Manage book inventory and availability

Issue & Return Management

* Record issue and return transactions
* Automatically generate due dates
* Calculate fines for overdue returns

Data Persistence

* Stores book records in files
* Maintains issue and return history
* Preserves data between program executions

Advantages

* User-friendly and efficient
* Reduces manual workload
* Faster record management
* Accurate tracking of books and users
* Secure and organized data storage

Future Enhancements

* Graphical User Interface (GUI)
* Database integration (MySQL/SQLite)
* User authentication and role management
* Online book reservation system
* Barcode and QR code support

Conclusion

This project demonstrates the practical implementation of data structures, file handling, and modular programming concepts in C while providing a complete solution for managing library operations efficiently.
